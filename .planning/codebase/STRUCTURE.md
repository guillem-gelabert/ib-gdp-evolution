## Directory Purposes

**`app/`:**
- Purpose: Nuxt 4 application source (Nuxt 4 defaults to `srcDir: "app/"`).
- Contains: `app.vue`, `pages/`, `components/`.
- Key files: `app/app.vue`, `app/pages/index.vue`.

**`app/components/`:**
- Purpose: Auto-imported Vue 3 SFCs — kebab-case tag names match file names.
- Contains: `line-chart.vue`, `story-page.vue`.
- Key files: `app/components/line-chart.vue`, `app/components/story-page.vue`.

**`app/pages/`:**
- Purpose: File-based routes; each `.vue` file becomes a route.
- Contains: `index.vue` only (single-page site).
- Key files: `app/pages/index.vue`.

**`public/`:**
- Purpose: Static assets served as-is from the site root.
- Contains: `favicon.ico`, `robots.txt`, `data/`.
- Key files: `public/data/balearic_gdp_pc.csv` (fetched by the app at `/data/balearic_gdp_pc.csv`).

**`public/data/`:**
- Purpose: Preprocessed CSVs for the browser (and working tables kept alongside them).
- Contains: Four CSVs; `balearic_gdp_pc.csv` is the only one currently read by the Vue code.
- Key files: `public/data/balearic_gdp_pc.csv`.

**`scripts/`:**
- Purpose: Offline Python ETL to chain-link Rosés-Wolf + Eurostat/INE data into the CSVs that feed the site.
- Contains: `extend_gdp.py`.
- Key files: `scripts/extend_gdp.py`.

**`.planning/codebase/`:**
- Purpose: GSD-generated codebase analysis documents (this file lives here).
- Contains: ARCHITECTURE.md, STRUCTURE.md, and other focus-area docs.

**`.nuxt/`:**
- Purpose: Generated Nuxt build output (tsconfig references, eslint config, dev manifests). Do not edit.
- Generated: Yes. Committed: No.

**`node_modules/`:**
- Purpose: pnpm-installed dependencies. Do not edit.
- Generated: Yes. Committed: No.

## Key File Locations

**Entry Points:**
- `app/app.vue`: Nuxt root SFC (`<NuxtPage />`).
- `app/pages/index.vue`: Home route rendering `<story-page />`.
- `scripts/extend_gdp.py`: Offline ETL CLI.

**Configuration:**
- `nuxt.config.ts`: Modules (`@nuxt/eslint`, `@nuxtjs/tailwindcss`), Google Fonts links, `vite.optimizeDeps.exclude: ["d3"]`, `compatibilityDate: "2025-07-15"`.
- `tailwind.config.ts`: Custom colors (`cream`, `ink`, `muted`, `accent`, `rule`, `footer-bg`) and font families (`headline`, `body`, `label`).
- `eslint.config.mjs`: `withNuxt()` from `.nuxt/eslint.config.mjs`.
- `tsconfig.json`: References `./.nuxt/tsconfig.*.json`.
- `package.json`: Scripts `dev` / `build` / `generate` / `preview` / `postinstall`.
- `pnpm-lock.yaml`, `pnpm-workspace.yaml`: pnpm is the package manager.

**Core Logic:**
- `app/components/story-page.vue`: Steps array, CSV fetch + parse, scrollama setup, sticky scroll-arrow geometry, page chrome (hero, prose, figure, outro, data-methods, footer).
- `app/components/line-chart.vue`: D3 SVG chart — scales, axes, line/point joins, tooltip, `GdpDataPoint` interface.

**Data:**
- `public/data/balearic_gdp_pc.csv`: Columns `year,gdp_pc,source,unit`; `source` is `RW` (1900–1990 decadal) or `INE` (2000–2024 annual, chain-linked).
- `public/data/balearic_ine_chainlinked_gdp_pc.csv`: Columns `series,year,value,unit,method`.
- `public/data/balearic_ine_gdp_pc_datalake.csv`: Columns `series,year,value,unit,source_note` (current-price euros, not plotted directly).
- `public/data/roses_wolf_selected_comparison.csv`: Columns `series,year,value,unit`.

**Testing:**
- None. No `*.test.*` / `*.spec.*` files, no Vitest/Jest config.

## Naming Conventions

**Files:**
- Vue SFCs: `kebab-case.vue` (`line-chart.vue`, `story-page.vue`, `index.vue`, `app.vue`). Template tag names mirror the filename (`<line-chart />`, `<story-page />`).
- Config: `*.config.ts` / `*.config.mjs` at repo root (`nuxt.config.ts`, `tailwind.config.ts`, `eslint.config.mjs`).
- CSVs: `snake_case.csv` under `public/data/`.
- Python: `snake_case.py` (`extend_gdp.py`).

**Directories:**
- All lowercase (`app`, `public`, `scripts`, `components`, `pages`, `data`).

**Code identifiers:**
- TypeScript types / interfaces: `PascalCase` (`GdpDataPoint`).
- Vue reactive refs / functions: `camelCase` (`activeStep`, `scrollArrowHeight`, `computeArrowStickyTop`).
- Module-scope D3 selections: `camelCase` (`xScale`, `linesG`, `tooltipBg`).
- Constants: `SCREAMING_SNAKE_CASE` (`WIDTH`, `HEIGHT`, `MARGIN`, `DURATION`, `LINE_COLOR`).
- Python: `snake_case` for functions/variables, `PascalCase` for dataclasses/exceptions (`PipelineError`, `CheckResult`), `SCREAMING_SNAKE_CASE` for module constants (`YEAR_MIN`, `ANCHOR_YEAR`).

**Tailwind tokens:**
- Semantic names from `tailwind.config.ts` (`bg-cream`, `text-ink`, `text-accent`, `border-rule-light`, `font-headline`, `font-body`, `font-label`). Prefer these over raw hex values.

## Where to Add New Code

**New Vue component:**
- File: `app/components/<kebab-name>.vue`. It is auto-imported — reference it as `<kebab-name />` in any template.
- Types consumed across components: export them from the component file and `import type { ... } from "./<component>.vue"` (see `GdpDataPoint`).

**New page / route:**
- File: `app/pages/<segment>.vue`. Keep `index.vue` as `/`; nested folders create nested routes.

**New static asset / dataset:**
- Put it under `public/` (served at site root). For data, use `public/data/<snake_case>.csv` with a stable header row; fetch via `useFetch("/data/<name>.csv", { server: false })` and parse with `d3.csvParse`.

**New data-pipeline step:**
- Extend `scripts/extend_gdp.py` (or add a sibling `scripts/<snake_name>.py`). Inputs resolve from `data/` then `public/data/` via `find_input_file`. Raise `PipelineError` on validation failures. Write outputs under `output/` (the script creates `output/` at runtime; it is not committed).

**New utility / composable:**
- No `composables/` or `utils/` directory exists yet. Create `app/composables/` for `useX()` composables (Nuxt auto-imports them) or `app/utils/` for pure helpers; keep files `kebab-case.ts`.

**Shared types:**
- Co-locate with the owning component (current pattern). If a type is truly cross-cutting, introduce `app/types/` and export from `.ts` files.

**New third-party module:**
- Add via `pnpm add <pkg>`. For Nuxt modules, register in `nuxt.config.ts` under `modules: [...]`.

**Styling:**
- Prefer Tailwind utilities with tokens from `tailwind.config.ts`. Extend the theme there rather than hardcoding new hex values in components.
